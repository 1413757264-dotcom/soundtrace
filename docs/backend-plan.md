# 音迹 SoundTrace — 后端能力全景

## P0 — 没有后端 App 就是空壳

| 功能 | 具体做什么 | 依赖的外部 API |
|------|-----------|---------------|
| **真实歌曲搜索** | 把 32 首 mock 替换成 Spotify/Apple Music 的 8000 万首曲库 | Spotify Web API |
| **采样溯源** | 查一首歌用了哪些采样、每段采样被哪些歌复用 | WhoSampled (需爬虫) |
| **BPM/调式/风格** | 每首歌自动返回 BPM、调式、能量、舞动性等音频特征 | Spotify Audio Features |
| **艺人/制作人数据** | 艺人详情、作品列表、关联艺人 | Spotify + MusicBrainz |

## P1 — 核心差异化

| 功能 | 具体做什么 | 备注 |
|------|-----------|------|
| **录音识曲** | 用户哼唱/外放录音 → 音频指纹 → 返回歌曲信息 | AcoustID / AudD |
| **制作人人脉图谱** | 基于真实 credit 数据构建协作网络 | MusicBrainz + Discogs |
| **采样对比音频** | 返回原采样老歌的 30s 预览 + 新歌对应片段 URL | Spotify Preview URL |
| **新歌实时入库** | 当天发布的歌，API 还没收录 → 后台队列下载+分析+入库 | yt-dlp + Demucs |

## P2 — 推送 + 智能

| 功能 | 具体做什么 |
|------|-----------|
| **采样更新推送** | "你收藏的采样被 XXX 新歌用了" → 推送通知 |
| **个性化推荐** | 基于浏览/收藏历史，推荐采样源头相近的歌曲 |
| **热门采样排行** | 实时统计被采样最多的老歌/艺人，生成排行榜 |
| **采样专题歌单** | 自动聚合"James Brown 被采样全集""2025 最佳采样说唱" |

## 数据工程层

| 功能 | 说明 |
|------|------|
| **音源分离管线** | Demucs/Spleeter 自动分离人声/鼓/贝斯/其他 → 每轨独立指纹比对 |
| **音频指纹入库** | Chromaprint 对每首歌的每个分离轨生成指纹 → 存 PostgreSQL |
| **批量对账任务** | "新入库 1000 首 → 逐首跑分离+指纹 → 比对已有指纹库 → 自动标记采样匹配" |
| **采样置信度模型** | 指纹匹配分 + 时域对齐分 + 人工标注 → 0-100% 置信度 |
| **YouTube 音频抓取** | yt-dlp 下载原采样老歌（仅音频）→ 生成指纹入库 |
| **冷启动爬虫** | 先把 WhoSampled 上排名前 1000 的采样关系全爬下来做种子数据 |

## 用户系统

| 功能 | 说明 |
|------|------|
| **账号体系** | 手机号/邮箱注册 + 微信/Apple/Google 第三方登录 |
| **个人主页** | 收藏的采样、浏览历史、贡献的采样标注、关注列表 |
| **采样收藏同步** | 多端同步收藏夹 + 收藏夹分享链接 |
| **关注系统** | 关注制作人/艺人 → 新采样推送 |
| **语音房** | 每首歌/每个采样开房间，爱好者在线拆解 beat |
| **采样标注贡献** | 独立音乐人/用户可提交采样来源标注 → 审核后入库 → 积分/声望 |

## 内容平台

| 功能 | 说明 |
|------|------|
| **音乐人入驻** | 上传原创 + 自行标注采样来源 → 被溯源时精准曝光 |
| **采样专题歌单** | 编辑/算法自动生成主题歌单 |
| **采样日报/周报** | "本周最热的 10 段新采样" → Push 推送 |
| **采样时间线** | 一段 1970 年老歌采样 → 历年各版本使用记录 |
| **采样对比投票** | "哪个版本用这段采样用得最好" → 社区投票 |

## 推荐与算法

| 功能 | 说明 |
|------|------|
| **协同过滤** | "收藏了采样 A 的人也收藏了采样 B" |
| **制作人向量化** | 基于合作网络做 embedding → 相似制作人推荐 |
| **采样风格迁移** | 音频 embedding 相似度 → 推荐类似质感的采样 |
| **新歌发现** | 每天抓取新发行 → 自动跑采样检测 → 推送 |

## 基础设施

| 功能 | 说明 |
|------|------|
| **CDN 音频分发** | 30s 采样预览片段的存储 + 全球 CDN 加速 |
| **API 限流 + 鉴权** | 免费层 100 次/天 → 付费层无限 |
| **搜索索引** | Elasticsearch 全文搜索 (歌曲/艺人/采样/制作人/风格/年代/BPM) |
| **图数据库** | Neo4j 存制作人-艺人-采样关系网络 → 图谱查询毫秒级 |
| **消息队列** | Redis/BullMQ 处理音频分析异步任务 |
| **监控告警** | Sentry + Prometheus + Grafana |
| **管理后台** | 审核采样标注、查看数据质量、手动修正采样关系 |

## 技术架构

```
┌──────────┐     ┌─────────────────┐     ┌──────────────┐
│  React   │────▶│  BFF (FastAPI)  │────▶│ Spotify API  │
│  Native  │     │  /api/search    │     │ WhoSampled   │
│  App     │◀────│  /api/samples   │◀────│ AcoustID     │
└──────────┘     │  /api/graph     │     │ MusicBrainz  │
                 │  /api/recognize │     │ YouTube Data │
                 └───────┬─────────┘     └──────────────┘
                         │
                 ┌───────┴─────────┐
                 │  PostgreSQL     │
                 │  + Redis 缓存    │
                 │  + S3 音频存储   │
                 │  + Elasticsearch │
                 │  + Neo4j 图谱    │
                 └─────────────────┘
```

## 建议实施顺序

1. **FastAPI BFF** — 聚合 Spotify + MusicBrainz 搜索 → 替换 mock
2. **PostgreSQL** — Song/Sample/Artist/Credit 表建库
3. **WhoSampled 爬虫** — 种子数据入库
4. **Redis 缓存** — API 响应缓存 + 限流
5. **音频指纹管线** — Demucs + Chromaprint
6. **用户系统** — 注册/登录/收藏
7. **Elasticsearch** — 全文搜索
8. **推送 + 推荐**
